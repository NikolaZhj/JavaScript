# 数字飞花令 — 游山玩水觅诗仙 期末大作业方案文档

---

## 一、背景

"飞花令"是中国古代文人饮酒时的一种雅致行令游戏，得名于唐代诗人韩愈《寒食》中"春城无处不飞花"的名句。传统飞花令要求参与者轮流背诵含有特定"令字"的诗词，规则简单却极度考验参与者的文学素养与临场反应能力。

在教育部大力推动"中华优秀传统文化进校园"、国家文物局推进"革命文物与数字化融合"的大背景下，本项目将传统飞花令进行数字化改造，以中文数字（一至万）作为令数核心，融合横版探索与回合制对战玩法，构建了一款游山玩水寻觅古代诗人的像素风 H5 游戏。用户在像素画风的山水世界中自由探索，邂逅李白、杜甫、苏轼、王维四位大诗人，以"飞花令"对决一较高下，在游戏中感受诗词文化的魅力。

---

## 二、设计目标

1. **文化传承**：通过探索+对战的游戏化体验，让用户接触并记忆大量中国传统诗词和成语，寓教于乐。
2. **沉浸探索**：构建 5000px 宽横版像素场景，融入王国新大陆风格的草地、河流、山峦、花草、飞鸟等元素，营造游山玩水的意境。
3. **名人互动**：设计李白(诗仙)、杜甫(诗圣)、苏轼(东坡)、王维(诗佛)四位 Boss，每位有独特的像素形象和难度梯度。
4. **对战趣味**：探索模式下走近诗人触发飞花令对战，支持人机 Boss 战和本地双人对战两种模式。
5. **信息可视化**：HP 血条、20秒倒计时、气泡对话展示词汇、小地图导航、胜率统计等。
6. **增强体验**：8bit 像素音效、粒子爆散特效、屏幕震动、气泡对话、走路动画等增强交互沉浸感。
7. **跨平台**：纯 H5 单文件实现，浏览器直接运行，PC 端和移动端均可。

## 三、主要内容

### 3.1 双模式设计

| 模式 | 说明 |
|------|------|
| 🏔 **游山玩水（探索+Boss战）** | ←→ 键移动角色，5000px 横版场景自由探索，走近诗人自动触发飞花令对战。击败后诗人变灰标识 ✓，全部击败触发通关彩蛋。 |
| 👥 **双人对战（PvP）** | 顶部按钮切换，两位玩家共用输入框轮流出招。红方玩家一 vs 蓝方玩家二，各自独立 HP 条。三局两胜制。 |

### 3.2 Boss 诗人系统

| 诗人 | 称号 | 位置 | HP | 攻击 | 主题色 |
|------|------|------|-----|------|--------|
| 🍶 李白 | 诗仙 | x=800 | 120 | 10 | 金色 |
| 📜 杜甫 | 诗圣 | x=1700 | 140 | 12 | 翠绿 |
| 🎋 苏轼 | 东坡居士 | x=2700 | 160 | 14 | 钴蓝 |
| 🏔 王维 | 诗佛 | x=3800 | 180 | 16 | 古铜 |

### 3.3 对战规则
- **令数系统**：随机抽取中文数字（一至万共14个）作为令数
- **出招规则**：输入含当前令数的成语或诗句，不能重复使用
- **伤害机制**：玩家每次 8-14 伤害，Boss 根据攻击力造成伤害
- **时间限制**：每回合 20 秒倒计时，超时判负
- **胜负制**：三局两胜（先赢 2 局者胜）

### 3.4 数据资源
- **成语数据库**：180+ 条含数字的中文成语（覆盖一至万）
- **诗句数据库**：48 条经典唐诗宋词名句
- **词汇总量**：Map 数据结构统一管理，O(1) 查询
- **Boss 数据**：四位诗人各有独立的名称、称号、属性、配色

### 3.5 场景视觉（王国新大陆像素风格）
- **天空**：蓝天渐变 + 白云 + 飞鸟 + 太阳光晕
- **山峦**：三层视差滚动（远山淡绿、中山翠绿+浅色层、近山深绿）
- **地面**：多层草地（深泥底+翠绿主层+渐变带+噪点纹理+草丛簇+小花）
- **小路**：石径铺路 + 散落鹅卵石
- **河流**：深水底+水面主体+动态水波纹理+高光闪烁+绿草岸边
- **建筑**：凉亭、石桥、柳树、竹林
- **角色**：像素块角色（头盔、身体、腰带、手臂、腿+走路动画）

## 四、创新创意

### 4.1 交互创新
- **横版探索+遭遇战**：打破传统对战游戏的固定画面，玩家可在山水场景中自由行走，邂逅诗人触发战斗，增加探索感和惊喜感。
- **气泡对话系统**：出招时角色头顶弹出白底黑字气泡，像漫画对话框一样展示诗句成语，清晰可读。
- **屏幕震动反馈**：受击和胜利时的屏幕震动，增强打击感。
- **提示按钮**：提供 💡 提示功能，自动帮玩家从词库中随机选一句含当前令数的诗句成语，降低上手难度。
- **小地图导航**：底部小地图实时显示玩家和诗人位置，已击败诗人显示灰色。

### 4.2 文化科普创新
- **名人堂 Boss 设计**：将李白、杜甫、苏轼、王维四位大诗人作为游戏 Boss，每位配有 emoji 图标和专属配色。击败后获得"诗卷"，寓教于乐。
- **词汇即武器**：成语和诗句本身就是攻击手段，玩家在对战中自然记忆大量传统文化词汇。
- **Boss 难度梯度**：按历史时间线排列诗人位置和难度（李白→杜甫→苏轼→王维），潜移默化传递文学史知识。

### 4.3 技术融合创新
- **王国新大陆像素美学**：纯 Canvas `fillRect` 像素块绘制所有场景元素，不使用任何图片资源，代码即美术。
- **Web Audio API 8bit 音效**：Square 波型合成走路、出招、受击、胜利等音效，不依赖音频文件。
- **双模式状态机**：探索模式(boss encounters) 和 对战模式(battle/PvP) 的无缝切换。
- **视差滚动**：远中近三层山脉分别以 0.2/0.4/0.7 倍速滚动，增强场景纵深感。

## 五、JavaScript 技术应用及框架库选择

### 5.1 技术选型

本项目选择**纯原生 JavaScript（Vanilla JS）+ HTML5 Canvas**实现，仅引入 Google Fonts 的 Press Start 2P 像素字体。不依赖任何第三方 JS 框架或库。

| 考虑因素 | 选型理由 |
|---------|---------|
| **性能需求** | 游戏需 60fps 持续渲染全屏像素场景 + 粒子 + 气泡 + 视差滚动，Canvas API 是最优方案 |
| **项目规模** | 单人独立完成，约 290 行代码，无需框架工程化支撑 |
| **部署简单** | 单文件 HTML 直接双击浏览器运行，无需构建/打包/npm |
| **学习价值** | 深入理解 Canvas 像素绘制、游戏循环、状态机管理等底层知识 |

### 5.2 核心技术应用

| 技术 | 应用场景 |
|------|---------|
| **HTML5 Canvas API** | 全游戏渲染：场景(天空/山峦/草地/河流/建筑)、角色、粒子、气泡、小地图 |
| **`ctx.fillRect` 像素绘制** | 所有视觉元素使用 `fillRect` 像素块构建，配合 `imageSmoothingEnabled = false` |
| **`ctx.createLinearGradient`** | 天空渐变、太阳光晕 |
| **`ctx.createRadialGradient`** | 太阳光晕效果 |
| **`requestAnimationFrame`** | 游戏主循环，基于 deltaTime 控制帧率稳定 |
| **`performance.now()`** | 时间戳计算 deltaTime，实现帧无关动画 |
| **ES6 Map** | 词汇→令数映射，O(1) 查询 |
| **ES6 Set** | 已用词汇去重检测 |
| **Web Audio API** | `AudioContext` + `OscillatorNode` + `GainNode` 合成 8bit 像素音效 |
| **CSS3 动画** | `@keyframes msgF` 消息弹出动画 |
| **响应式设计** | `@media(max-width:600px)` 适配移动端 |
| **DOM 事件** | `keydown/keyup` 角色移动、`click` 按钮交互、`touchmove` 防滚动 |
| **Google Fonts** | Press Start 2P 像素字体（唯一外部依赖） |

### 5.3 数据结构设计

```
allWordsMap: Map<词汇, 令数>              → 词汇库快速查询
numbers: Array<中文数字>                  → 令数候选池
bosses: Array<Boss对象>                   → 诗人数据(名称/位置/属性/配色)
state: Object                             → 集中式状态管理
  ├── mode: 'explore'|'battle'|'pvp'     → 三模式状态机
  ├── playerX/playerVX/cameraX           → 探索模式
  ├── currentBoss/currentNumber/hp[]      → 对战模式
  ├── bubbles[]                          → 气泡队列
  ├── particles[]                        → 粒子队列
  ├── floatPixels[]                      → 飘浮字队列
  └── flowers[]                          → 花草数据
```

### 5.4 游戏循环架构

```
requestAnimationFrame(loop)
  ├── 计算 deltaTime
  ├── 探索模式更新：键盘输入→角色移动→相机跟随→Boss检测
  ├── 粒子更新：位置/速度/生命周期
  ├── 气泡更新：位置/透明度
  └── 绘制：clearRect → 场景(天空/云/鸟/太阳/山/地/路/河/建筑) → Boss/角色 → 飘浮字 → 小地图 → HP/计时器 → 气泡 → 粒子
```

## 六、总结

本项目以"游山玩水觅诗仙"为主题，将中华优秀传统文化中的诗词飞花令与横版探索RPG玩法深度融合。在技术上采用纯原生 HTML5 Canvas 像素渲染 + Web Audio API 8bit 音效，构建了一个 5000px 宽、具有《王国：新大陆》美学风格的像素世界。在内容上以李白、杜甫、苏轼、王维四位大诗人为 Boss，让玩家在"以文会友"的对战中感受传统文化的魅力。在设计上提供探索模式（冒险+挑战诗人）和双人对战模式，并通过气泡对话、粒子特效、屏幕震动、💡 提示按钮等增强交互体验。

本项目实现了 H5 技术载体、中华优秀传统文化主题、数字资源运用、技术艺术融合、信息可视化展示、交互创新增强体验等期末大作业的全部核心要求。
